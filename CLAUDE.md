# CLAUDE.md

Project guidance for Claude Code when working in this repo.

## What this repo is

**Foyer** — a conversational booking agent for local-service businesses
(plumber, cleaner, dog walker). SMS + web chat intake, 15-minute slot hold
backed by a Postgres exclusion constraint, human-confirm gate before any
calendar lock. Buyer is the small-shop owner whose Upwork listing reads
"AI booking bot for my service business."

The wedge: every booking-bot demo on the market either auto-books
(ghost-show factory) or chats without integrating a calendar. Foyer holds a
tentative slot for 15 minutes, requests human approval before locking, and
refuses to commit outside its configured scope.

## Conventions

- **No AI attribution anywhere.** No `Co-Authored-By`, no "generated by
  Claude" in commits, code, comments, README, deploy logs, anything.
- **Postgres 16, not MySQL.** `btree_gist` extension is required for the
  slot-hold + bookings exclusion constraints (the load-bearing guard
  against double-booking). Schema is `foyer`.
- **ULIDs (`CHAR(26)`)** for all user-facing primary keys via
  `symfony/uid`. Don't mix with auto-increment IDs on client-visible
  resources.
- **OpenAPI is the source of truth** for `/v1/*` endpoints. Implement
  controllers to match `openapi/spec.yaml`, not the other way around.
  Spectral lint + contract tests gate CI.
- **Internal API uses HMAC.** Every request between Laravel and the
  FastAPI agent worker carries `X-Foyer-Internal-Sig:
  hmac_sha256(shared_secret, body)`. Internal API binds to 127.0.0.1 only;
  reject 403 otherwise. Documented in `docs/architecture/internal-api.md`.
- **Idempotency-Key header required** on every owner-facing mutating
  endpoint that has a side-effect outside Foyer (`POST /v1/bookings/:id/
  confirm`, `POST /v1/bookings/:id/reject`). Backed by `idempotency_keys`
  table, 24h TTL.
- **RFC 7807 problem details** for error responses
  (`application/problem+json`). No `{"error": "..."}` shapes.
- **STOP / START / HELP win, no override.** Twilio's mandatory keywords
  are enforced before any agent dispatch. Consent state is keyed on
  `(customer_phone_e164, twilio_number_e164)`.
- **Quiet hours respected on every outbound.** Default 21:00–08:00
  customer-local; per-business override.

## Three-process service contract

The single most failure-prone seam. Read
`docs/architecture/internal-api.md` and `docs/architecture/why-fastapi.md`
before adding cross-process code.

- **Laravel** owns the DB schema, all migrations, all writes. Sanctum-
  auth for owner-facing endpoints. Sole producer of queue jobs.
- **FastAPI agent worker** consumes `AgentTurn` jobs from a Redis queue
  (Laravel-side enqueue) and POSTs results back to the internal API.
  Never opens a Postgres connection directly.
- **Twilio outbound** is a PHP Horizon queue worker, not a separate
  process. One concern, one runtime.

The `AgentTurn` job payload is JSON-Schema'd at
`infra/contracts/agent-turn.schema.json` and validated on enqueue and
dequeue. PHP + Python codegen from the same schema; CI gates parity.

## Webhook ingestion — queue-fast-ack pattern

`POST /v1/sms/inbound` and `POST /v1/web/inbound` MUST return `200`
within 500ms:

1. Validate `X-Twilio-Signature` (SMS) or per-session token (web).
   Fail-closed.
2. `INSERT ... ON CONFLICT (external_id) DO NOTHING` on `messages`.
   Idempotent on Twilio `MessageSid` or web `widget_message_id` (UUIDv7).
3. If insert returned no row (duplicate), return 200. No agent dispatch.
4. Dispatch `AgentTurn(conversation_id, message_id)` to Redis. Return 200.

Octane is dropped — the ~30ms win is irrelevant against LLM round-trip.
Plain php-fpm avoids singleton-state foot-guns.

## Deployment

- **Local builds, no CI deploys.** Composer is not installed on EC2.
  Vendor ships in the release tarball.
- **Atomic releases** for Laravel (`scripts/deploy/deploy.cjs`). Post-
  symlink-swap, run `sudo systemctl reload php8.3-fpm` so OPcache picks
  up the new release. Apache itself does not need a reload.
- **rsync + PM2 reload** for the Next.js `web/` half.
- **Postgres 16** on the EC2 host. Schema `foyer`, owner `foyer`, password
  in `/var/www/foyer/shared/.env` (gitignored).
- **Supervisor entries** for Horizon (`/etc/supervisor/conf.d/foyer-
  horizon.conf`) and the FastAPI agent worker (`foyer-agent-worker.conf`).

## Things to avoid

- Don't run `php artisan migrate:fresh` against production. There is no
  staging DB yet.
- Don't try to mock the exclusion constraint in tests — exercise it
  against a real Postgres test database. Mocks defeat the whole point.
- Don't bypass the queue-fast-ack pattern in tests. Webhook handlers must
  never block on LLM calls.
- Don't commit `.env`, `.env.deployment`, or anything under `.local/`.
- Don't add a `current_phase` column to `conversations` — it's derived
  from the last message's `phase`. Tests assert this.
- Don't relax the EXCLUDE constraints to ON CONFLICT-style upserts. The
  double-booking guard is the constraint; tests run concurrent inserts
  against a real Postgres and assert exactly one wins.
